# 第3课 — 创建动态内容

*您的挑战：*&#x200B;您需要确保仅要求尚未安装该应用程序的客户安装该应用程序。 已安装该应用程序的任何人应该只收到一条常规欢迎消息。 您不希望使历程流变得更加复杂。 

*解决方案*：动态内容允许您根据条件规则调整消息的内容，条件规则可由配置文件属性、上下文事件或受众组成。 

在本练习中，您以将动态内容应用于短信为例，但它可以应用于所有渠道。
